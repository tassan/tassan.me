What to do when you just need to update your master/main branch and don’t want to lose your work in progress?

Nov 9 2021

Imagine you're on branch A, but you need to update your master/main branch, whatever your motive may be, and then, you want to get back to your working branch. Well, this is what you should do:

On Branch A:

git stash
git switch master/main

On Branch Master/Main:

git pull
git switch branch-a

Now here is the trick, if you do a git status, git will problaby say this:

On branch branch-a
Your branch is up to date with 'origin/branch-a'. nothing to commit, working tree clean

Which is true, for instance, if you use: git log --oneline, you'll see your last commit listed first, as if any of your previous work was lost. But, don't panic. Now you just have to “unstash” your work using:

git stash pop

And git will prompt all your work items, that you thought you've lost.
